Xenith workspace: list open windows first to understand user context.

Commands:
    xenith list              — list windows (do this first)
    xenith read <id> body    — read window content
    xenith create name       — create window (save the returned ID)
    xenith write <id> body text — write to window (single line)
    xenith append <id> text  — append to window body
    xenith ctl <id> name new — rename a window
    xenith status <id> ok    — set status: ok/warn/error/info

CRITICAL: Multi-line content REQUIRES heredoc:
    xenith write 4 body <<EOF
    Line one
    Line two
    EOF
Without <<EOF, only the first line is written.
